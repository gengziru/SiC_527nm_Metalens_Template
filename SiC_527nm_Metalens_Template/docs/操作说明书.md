# 超透镜设计基础模板和操作说明书

这份模板把 527 nm SiC 单波长超透镜设计流程整理成三个部分：

- `Unit`：超原子单元扫参，得到相位库。
- `Metalens`：整片超透镜相位设计、半径匹配、FDTD 建模和后处理。
- `GDS`：把已经生成的半径矩阵转换为二维 GDS 版图。

核心思想是：先用单元扫参建立“半径 -> 相位/透过率”的超原子库，再根据目标聚焦相位为每个晶格点选择最接近的半径，随后在 FDTD 中搭建整片超透镜并验证焦点；当半径矩阵确认无误后，再导出 GDS 版图用于后续加工沟通。

## 一、Unit：超原子相位库

### 操作目的

得到当前材料、波长、周期、高度下的相位库：

- `phix_527.mat`
- `Tx_527.mat`

这两个文件是后续整片超透镜设计的基础。如果波长、材料、周期、高度发生变化，必须重新扫参。

### 当前基础参数

- 波长：`527 nm`
- 材料：SiC，近似折射率 `n = 2.67`
- 周期：`224 nm`
- 圆柱高度：`600 nm`
- 半径范围：`44 nm` 到 `92 nm`
- 半径步长：`0.5 nm`
- 扫描点数：`97`

### 具体操作

1. 在 FDTD 中打开单元模型。
2. 确认 sweep 名称是 `radius`。
3. 运行 radius sweep。
4. 运行 `Unit/unit_sweep_template.lsf`，导出 `phix_527.mat` 和 `Tx_527.mat`。
5. 在 MATLAB 中运行 `Unit/plot_unit_phase_library_template.m`，检查相位覆盖和透过率曲线。

### 判断结果是否正常

- 相位最好接近覆盖一个完整 `2*pi`。
- 透过率不要出现大面积接近 0。
- `phix_527.mat` 和 `Tx_527.mat` 的数据点数必须和半径扫描点数一致。

## 二、Metalens：整片超透镜设计

### 操作目的

把目标聚焦相位转换为每个晶格点的纳米柱半径，并自动生成 FDTD 建模脚本。

### 设计公式

双曲相位：

```text
phi = -2*pi/lambda * (sqrt(x^2 + y^2 + f^2) - f)
```

二次相位：

```text
phi = -2*pi/lambda * (x^2 + y^2)/(2*f)
```

双曲相位更精确，二次相位是近轴近似。小尺寸、低 NA、正入射时两者图像会很像；离轴或更大 NA 时差异更明显。

### 具体操作

1. 打开 `Metalens/generate_target_radius_template.m`。
2. 修改 `cfg.project_tag`，例如：
   - `hyperbolic_527_small`
   - `quadratic_527_small`
   - `hyperbolic_527_small_angle10`
3. 修改 `cfg.phase_model`：
   - `hyperbolic`
   - `quadratic`
4. 修改 `cfg.theta_deg`：
   - `0`：正入射
   - `10`：10 度离轴入射
5. 确认 `cfg.unit_dir = fullfile('..','Unit')` 可以找到相位库。
6. 在 MATLAB 中运行脚本。
7. 在 Lumerical 中运行生成的 `structure_lens_项目名.lsf`。
8. 点击 `Run` 运行 FDTD。
9. 仿真结束后运行后处理脚本。
10. 如果需要加工版图，再进入 `GDS` 文件夹生成 GDS。

### MATLAB 脚本输出

`generate_target_radius_template.m` 会生成：

- `target_radius_项目名.mat`：完整设计数据，适合 MATLAB 检查。
- `target_radius_项目名_fdtd.mat`：简化数值文件，作为备用。
- `target_radius_项目名.csv`：半径矩阵 CSV，作为备用。
- `structure_lens_项目名.lsf`：嵌入式 FDTD 建模脚本。
- `Metalens_项目名.fsp`：运行 LSF 后自动保存的 FDTD 工程。

## 三、GDS：版图生成

### 操作目的

把 `Metalens` 阶段生成的 `target_radius_项目名.mat` 转换为可在 KLayout 等版图软件中打开的 `.gds` 文件。

GDS 生成脚本不重新做相位匹配，而是直接读取已经生成的：

- `target_radius`：每个晶格点的实际半径。
- `target_index`：每个晶格点对应的超原子库索引。
- `radius_list_m`：相位库半径列表。
- `x_mask` / `y_mask`：晶格坐标。

这样可以保证 GDS 版图和 MATLAB/FDTD 使用的半径矩阵严格对应。

### 具体操作

1. 确认已经运行 `Metalens/generate_target_radius_template.m`。
2. 打开 `GDS/generate_gds_from_target_radius_template.m`。
3. 修改 `cfg.project_tag`，使其和 `Metalens` 脚本中的 `cfg.project_tag` 一致。
4. 如果半径矩阵不在默认位置，修改 `cfg.source_mat`。
5. 根据加工平台要求修改 `cfg.layer_id` 和 `cfg.datatype`。
6. 在 MATLAB 中进入 `GDS` 文件夹并运行脚本。
7. 用 KLayout 打开输出的 `metalens_项目名_inverse.gds`。

### 输出文件

- `metalens_项目名_inverse.gds`：反色 GDS 版图。
- `metalens_项目名_GDS_info.mat`：记录输入 MAT 路径、层号、周期、cell 命名、设计参数等信息。

### 默认反色版图含义

当前模板默认输出反色版图：

```text
曝光区域 = 单元方形背景
保护区域 = 中间圆形纳米柱 footprint
```

GDS 层级包括：

- `TOP`：顶层 cell。
- `METALENS_TOP`：整片超透镜排布 cell。
- `INV001` 到 `INVxxx`：不同半径对应的反色单元 cell。
- `CLEAR_CELL`：完整方形曝光单元，预留备用。

如果加工平台需要正色圆柱版图，需要在 GDS 脚本中把反色单元改为圆形 pillar polygon。

## 四、FDTD 建模脚本

### 操作目的

在 Lumerical 中自动建立：

- SiC 基底
- 纳米柱阵列
- FDTD 仿真区域
- TFSF 光源
- 近场 monitor

### 入射角由哪里控制

在生成的 `structure_lens_项目名.lsf` 中，入射角由 source 的属性控制：

```lsf
set("angle theta",theta_deg);
set("angle phi",phi_deg);
```

`theta_deg = 10` 表示在 x-z 平面内 10 度斜入射。理想情况下，焦点横向偏移约为：

```text
x_shift = f*sin(theta)
```

例如 `f = 10 um`，`theta = 10 deg`，则偏移约为 `1.736 um`。

## 五、后处理

### 远场图

运行 `Metalens/farfield_template.lsf`，得到：

- `Exz_项目名.jpg`：x-z 传播图，用来看实际聚焦位置和倾斜传播。
- `Exy_项目名.jpg`：焦平面 x-y 图，用来看焦斑形状。

### 定量检查

运行 `Metalens/focus_metrics_template.lsf`，得到：

- `focus_line_x_项目名.jpg`
- `focus_line_y_项目名.jpg`
- `focus_line_z_项目名.jpg`
- `focus_metrics_项目名.mat`

重点看：

- `z_focus`：实际焦点 z 位置。
- `FWHM_x`：x 方向半高全宽。
- `FWHM_y`：y 方向半高全宽。

### 初步效率

运行 `Metalens/focus_efficiency_template.lsf`，得到：

- `focus_efficiency_项目名.mat`

其中：

- `T_monitor`：通过 monitor 的透射比例。
- `encircled_transmitted_fraction`：焦斑桶内占已透射能量的比例。
- `focusing_efficiency`：二者乘积。

这一步主要用于同一口径、同一后处理定义下的相对比较。

## 七、常见问题

### 1. 为什么不用早期 `structure_lens.lsf`

早期脚本通常包含旧波长、旧材料、旧周期等设置，不能直接用于 527 nm SiC。这里的模板已经把参数统一为当前 527 nm SiC 示例流程。

### 2. 为什么生成嵌入式 LSF

之前遇到过 `matlabload cannot open file`，可能和中文路径、MAT 文件内容、文件版本有关。嵌入式 LSF 把 `target_radius` 直接写进脚本，最稳妥。

### 3. FDTD Run 很慢正常吗

正常。即使是小模型也有一千多根柱子和 3D FDTD 区域。进度条消失、对象树切到 Analysis、有 monitor 结果时，通常表示仿真完成。

### 4. 正入射下 hyperbolic 和 quadratic 为什么很像

因为小模型 NA 不高，二次相位是双曲相位的近轴近似；再加上图像归一化，差异会被压缩。离轴或更大口径时差异更明显。

### 5. 改参数时最容易漏掉什么

- 改波长时忘记重新做 Unit 扫参。
- 改半径库范围但没同步 `radius_start_m` 和 `radius_stop_m`。
- 后处理里的 `theta_deg` 和建模脚本里的 `theta_deg` 不一致。
- 后处理 `z` 范围没有覆盖实际焦点。
- FDTD 还没 Run 完就运行 farfield 后处理。
- 生成 GDS 时 `cfg.project_tag` 和 `target_radius_项目名.mat` 文件名不一致。
- 生成 GDS 前没有确认反色/正色是否符合加工平台要求。

### 6. GDS 打不开或版图尺寸不对怎么办

- 先确认 MATLAB 写 GDS 的过程已经正常结束，没有中途停止。
- 用 KLayout 打开时检查顶层 cell 是否为 `TOP`。
- 用 Ruler 量测主图案直径，应约等于 `2*R_eff`。
- 检查 `cfg.source_mat` 是否指向当前设计的半径矩阵，而不是旧项目文件。
- 检查 `cfg.layer_id` / `cfg.datatype` 是否符合加工或查看软件要求。

## 八、建议的新项目命名方式

建议以后每个设计新建一个项目文件夹，例如：

```text
Metalens_my_design/
  Unit/
  Metalens/
  GDS/
```

项目标签建议写清楚相位类型、波长、尺寸、角度：

```text
hyperbolic_527_small
quadratic_527_small
hyperbolic_527_R20um_f40um_angle10
```

这样后处理图片和 `.mat` 文件不会混在一起。
